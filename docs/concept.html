DEAR BODY
Summary Concept Development Dev Diary Reflection Prototype

Project Idea

Dear Body is a poetic, blog-based web application where users can write anonymous letters to their body or to specific body parts. The act of writing is framed as a form of self-address, care, and emotional reflection.

By removing public identity and usernames, the platform encourages vulnerability without performance. Letters are not written for others, but exist as quiet, personal statements within a shared space.

Motivation & Background

The project is inspired by zines, diaries, and feminist publishing practices that treat writing as an intimate and political act. Rather than focusing on productivity, optimisation, or visibility, Dear Body explores digital spaces as sites of care.

Writing to the body allows users to externalise thoughts, emotions, and tensions that are often difficult to articulate. The web application functions as a container for these expressions, rather than a system that evaluates or ranks them.

User Flow

The application distinguishes between reading and writing:

  • Visitors can read letters without creating an account
  • Writing letters and resonating with others requires login
  • All letters are published anonymously
  • Logged-in users can view their own letters on their profile page

This structure lowers the barrier to engagement while preserving a sense of intentionality around participation.

Design Approach

Visually, the project combines softness and rawness. A bold pink background, red typography, pixel fonts, and scanned textures reference both digital nostalgia and handmade publishing.

The interface avoids dense information layouts. Instead, it prioritises space, rhythm, and legibility, allowing text to remain the central element.

MVP Scope

Frontend

  • Login and registration flow
  • Form to write a letter (authentication required)
  • Public feed displaying all letters
  • Profile page showing the user’s own letters

Backend

  • Authentication endpoints (register, login)
  • CRUD operations for letters
  • Middleware to protect writing and resonating routes

Database

  • Users table (authentication data)
  • Letters table (content, body reference, timestamp)
